Sobre la API REST desarrollada:


La finalidad de todo esto es conocer, aprender y formarse en el uso de tecnologías como Nodejs o Angular, y el tratamiento de datos en tiempo real. Los datos estadísticos que se procesan en el Front-End han sido sacados de la wikipedia, concretamente de aquí.

A la hora de hacer las integraciones con las diferentes apis externas y los distintos widgets, no se ha tenido en cuenta la concordancia de los datos integrados, es decir, en una misma gráfica se pueden comparar, por ejemplo, el número de denuncias por violación en Europa con la puntuación de las películas del estudio Ghibli. No tiene sentido semántico, pero repito que la finalidad es aprender a integrar datos externos, no a que todos los datos que se comparen entre sí tengan relación.

Todos los datos que crees o borres en el Front-end también serán creados/borrados en las gráficas

Todo el código de esta aplicación la podéis encontrar en GitHub





Funcionamiento de la aplicación:


Para este trabajo he utilizado:



  • Nodejs

    Para el apartado del back-end, junto con paquetes como Express, body-parser, path... Todo para el tratamiento de los datos


  • AngularJs

    Para el Front-end y las distintas vistas (usando angular-route para ello)


  • JavaScript

    Tanto en el Front-end como en el Back-end


  • MongoDB

    Como base de datos para la aplicación desarrollada


  • BootsTrap

    Para algunos elementos estéticos, tales como botones/tablas...


  • Materialize

    Igual que bootstrap, para que la aplicación tenga un diseño visual más atractivo. El menú principal está hecho utilizando Materialize por ejemplo


  • POSTMAN

    Para la realización de tests sobre la api rest (get, post, put, delete y búsquedas principalmente). En el apartado "Docs" del menú principal podemos ver con más detalle dichos tests


  • Protactorjs

    Pequeños test sobre la interfaz de la api para comprobar su correcto funcionamiento. En GitHub, los tests se encuentran dentro de la carpeta "test"
